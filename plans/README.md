## Repository-wide execution gates

Every selected plan carries these project requirements:

1. Before editing, enumerate all affected callers and inspect the possible value
   types on every call path.
2. State implementation assumptions in three lines or fewer.
3. Add the smallest end-to-end regression test first and demonstrate that it
   fails for the expected reason. If no current observable failure can be
   produced, stop and report instead of performing a behavior-preserving
   refactor on passing characterization tests alone.
4. After every source-file edit, run `moon check` before editing another source
   file; stop and fix any diagnostic immediately.
5. Finish with focused tests, `moon test`, `moon info` plus generated-interface
   diff review, `moon fmt`, and a final `moon check`, along with plan-specific
   gates.
