# Array iteration model implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Recover the Array/prototype iteration-model sub-cohort (~242 strict / ~253 non-strict test262 failures) by making `has_array_like_element`, `get_array_like_element_interp`, and `Interpreter::get_property` honor own descriptors, the `data.holes` set, and the Array prototype chain — plus close hole-lifecycle gaps so the `holes` invariant ("`i ∈ data.holes` iff there is no own property at `str(i)`") holds.

**Architecture:** Three phases that land in order: (1) clear-on-mutation foundation in `array_like.mbt` and `property_set.mbt` restores the holes invariant for already-marked holes, (2) a private `array_index_lookup_result` helper in `array_like.mbt` plus three read-site rewrites give the iteration helpers and `Interpreter::get_property` spec-correct ordinary `[[Get]]` / `[[HasProperty]]` semantics, (3) mark-on-create foundation in `Array(n)`, `set_array_like_length`, `create_data_property_or_throw`, and `array_define_own_property` lets the new readers see previously-invisible holes. The split is deliberate: Phase 1 is invariant-restoring and should not regress; Phase 2 produces most of the test262 recovery; Phase 3 may surface pre-existing reader bugs and is per-site revertible.

**Tech Stack:** MoonBit interpreter (`moon check`, `moon test`, `moon fmt`, `moon info`). Source spec: [docs/superpowers/specs/2026-05-16-array-iteration-model-design.md](../specs/2026-05-16-array-iteration-model-design.md) v5.

```moonbit
///|
/// Classification of an array-indexed lookup per ES2024 §10.1.5
/// OrdinaryGetOwnProperty. The `Present` variant subsumes both "dense
/// element, no descriptor" and "own data descriptor" — because data
/// descriptors store their value in `data.elements[i]`, not in the
/// descriptor itself.
priv enum ArrayIndexHit {
  Present(Value)
  Hole
  OutOfRange
  OwnAccessor(Value?)
}

///|
/// Classify an array-indexed lookup without invoking any getter or
/// walking the prototype chain. Caller decides what to do with each case.
fn array_index_lookup_result(data : ArrayData, i : Int) -> ArrayIndexHit {
  let key = i.to_string()
  match data.bag.descriptors.get(key) {
    Some(desc) =>
      if desc.is_accessor {
        OwnAccessor(desc.getter)
      } else if i >= 0 && i < data.elements.length() {
        Present(data.elements[i])
      } else {
        OutOfRange
      }
    None =>
      if i < 0 {
        OutOfRange
      } else if data.holes.contains(i) {
        Hole
      } else if i < data.elements.length() {
        Present(data.elements[i])
      } else {
        OutOfRange
      }
  }
}
```
