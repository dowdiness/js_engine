# Host Internal Slots Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give embedders an opaque `HostSlotKey` + get/set/delete/has API so host bookkeeping on JS objects is invisible to user JavaScript (#518).

**Architecture:** Add `PropertyBag.host_slots : Map[Int, Value]` beside engine `internal_slots`. Keys are opaque `HostSlotKey` values allocated by `HostSlotKey::reserve()` (package-instance `Ref[Int]` counter, classified for the architecture-state audit, abort on exhaustion). Accessors live in new `interpreter/runtime/host_slots.mbt`; engine `InternalSlotKey` is unchanged. The `PropertyBag` field add and all bag-literal migrations are one atomic edit wave.

**Tech Stack:** MoonBit, `NEW_MOON_MOD=0 moon` CLI

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-11-host-internal-slots-design.md` (Approved)
- Always prefix moon commands: `NEW_MOON_MOD=0 moon …`
- **Schema-migration exception:** Adding `PropertyBag.host_slots` and updating every explicit bag literal is **one atomic multi-file edit**. Do not run `moon check` after editing only `value.mbt` — that leaves the tree uncompilable by design. Edit the field, `host_slots.mbt`, classification JSON, and all literal sites together, then run `moon check` once. For all other tasks, keep the usual “fix before next file” discipline.
- Do not re-export through root `@js_engine` facade in this slice
- Do not extend `InternalSlotKey` or share one map with engine slots
- `PropertyBag` stays `pub(all)` — boundary is API convention, not field opacity
- `reserve()` must never wrap; abort on exhaustion
- Module-level `host_slot_id_counter : Ref[Int]` is intentional process/package-instance state: classify it in `scripts/architecture_state_classified_mutable_state.json` and verify with `make architecture-state-audit` (do not put the counter on `Interpreter` — that would change the approved uniqueness contract)
- No test262 changes expected
- Before final commit: `NEW_MOON_MOD=0 moon info && NEW_MOON_MOD=0 moon fmt`, then `make check`, `NEW_MOON_MOD=0 moon test`, and `make architecture-state-audit`
- Stage only the exact paths this work changed — never `git add -u`

**2b. Create `interpreter/runtime/host_slots.mbt`:**

```moonbit
///| Embedder-owned host slots (#518). Separate from engine `internal_slots`.

///|
let host_slot_id_counter : Ref[Int] = { val: 0 }

///|
/// Embedder-owned slot identity. Construct only via `HostSlotKey::reserve`.
pub(all) struct HostSlotKey {
  priv id : Int
} derive(Eq, Hash, Debug)

///|
/// Allocate a unique key within this loaded runtime package instance.
/// Aborts if the allocator is exhausted — never wraps.
pub fn HostSlotKey::reserve() -> HostSlotKey {
  let id = host_slot_id_counter.val
  // Last valid id is 0x7FFFFFFE; refuse to wrap into negatives / reuse.
  guard id < 0x7FFFFFFF else { abort("HostSlotKey::reserve: exhausted") }
  host_slot_id_counter.val = id + 1
  { id, }
}

///|
pub fn PropertyBag::get_host_slot(self : PropertyBag, key : HostSlotKey) -> Value? {
  self.host_slots.get(key.id)
}

///|
pub fn PropertyBag::set_host_slot(
  self : PropertyBag,
  key : HostSlotKey,
  value : Value,
) -> Unit {
  self.host_slots[key.id] = value
}

///|
pub fn PropertyBag::delete_host_slot(self : PropertyBag, key : HostSlotKey) -> Unit {
  self.host_slots.remove(key.id)
}

///|
pub fn PropertyBag::has_host_slot(self : PropertyBag, key : HostSlotKey) -> Bool {
  match self.host_slots.get(key.id) {
    Some(_) => true
    None => false
  }
}

///|
pub fn get_host_slot(data : ObjectData, key : HostSlotKey) -> Value? {
  data.bag.get_host_slot(key)
}

///|
pub fn set_host_slot(data : ObjectData, key : HostSlotKey, value : Value) -> Unit {
  data.bag.set_host_slot(key, value)
}

///|
pub fn delete_host_slot(data : ObjectData, key : HostSlotKey) -> Unit {
  data.bag.delete_host_slot(key)
}

///|
pub fn has_host_slot(data : ObjectData, key : HostSlotKey) -> Bool {
  data.bag.has_host_slot(key)
}
```

Note: `key.id` is readable here because `host_slots.mbt` is the same package as the `priv` field. External packages must use the accessors.

**2c. Classify the module-level counter** in `scripts/architecture_state_classified_mutable_state.json` (replace `{}`):

```json

  "interpreter/runtime/host_slots.mbt:host_slot_id_counter": "Process/package-instance allocator for HostSlotKey::reserve (#518). Embedder slot identities are intentionally shared across interpreters in one loaded runtime package (approved design: not per-realm/per-interpreter). Monotonic Int; abort on exhaustion, never wrap. ```moonbit
///|
test "host slot is invisible to Object.keys for-in and JSON.stringify" {
  let interp = new_interpreter()
  // Ordinary property that must remain visible and unchanged beside the host slot.
  let prog = @parser.parse(
    #|var o = { visible: 1, __crater_nid: 7 };
  )
  let _ = interp.run(prog.stmts)
  let o = interp.global.get("o")
  let key = @runtime.HostSlotKey::reserve()
  match o {
    // Host bookkeeping value deliberately differs from the visible __crater_nid.
    Object(data) => @runtime.set_host_slot(data, key, Number(99.0))
    _ => fail("expected Object for o")
  }
  let check = @parser.parse(
    #|var keys = Object.keys(o).sort().join(",");
    #|var enumed = [];
    #|for (var k in o) enumed.push(k);
    #|enumed.sort();
    #|var json = JSON.stringify(o);
    #|var nid = o.__crater_nid;
    #|var missing = o.host_secret;
    #|console.log(keys);
    #|console.log(enumed.join(","));
    #|console.log(json);
    #|console.log(String(nid));
    #|console.log(typeof missing);
  )
  let _ = interp.run(check.stmts)
  json_inspect(interp.host.output, content=[
    "__crater_nid,visible",
    "__crater_nid,visible",
    "{\"visible\":1,\"__crater_nid\":7}",
    "7",
    "undefined",
  ])
}
```

Notes:

- `__crater_nid` is the colliding ordinary property; after attaching the host slot, JS must still read `7` and enumerate/stringify that property.
- Host slot value `99` must not appear in keys, enumeration, JSON, or property reads.
- If `JSON.stringify` key order differs, adjust the snapshot to the engine’s actual serialization **without** dropping `__crater_nid` or inventing host fields.
- If `Object.keys(…).sort` is unavailable, sort in the assertion helper or compare a known unordered-safe encoding — but keep both property names in the expected set.
